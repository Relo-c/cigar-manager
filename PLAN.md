# Cigar Manager 2.0 产品策划与实施计划

> 产品方向：操作优先型雪茄库存管理 PWA
> 目标设备：iPhone Safari / 添加到主屏幕
> 当前阶段：Phase 1「导航与视觉基础」已完成
> 视觉目标：Product Design 方案 3「Daily Steward」

## 一、最高优先级协作准则

1. 在执行任何操作前，必须先通过提问确认需求；只有在对用户需求的理解达到至少 95% 把握后，才能执行相应操作。
2. 问题必须一个一个地问。每次只提出一个问题，收到用户回答后，才能提出下一个问题。
3. “任何操作”包括读取、搜索、检查、修改、删除、运行命令、发布及影响外部状态的操作。

## 二、产品定位

Cigar Manager 是一款面向 iPhone 的中文雪茄库存管理 PWA。

第一阶段服务单用户收藏者，数据保存在设备本地并支持离线使用；产品架构为未来多用户、云同步和多设备协作预留扩展空间。

核心体验目标：

> 用户打开应用后，10 秒内知道库存现状，3 步内完成一次库存操作。

## 三、设计原则

### 3.1 操作优先

首页围绕四个高频动作设计：

- 入原盒
- 入散支
- 出库
- 拆盒

### 3.2 信息克制

- 首页只展示最关键的库存数字与当天动态。
- 完整库存放入库存页。
- 完整历史放入操作记录页。
- 数据工具和密码放入设置页。

### 3.3 结果明确

每次操作都要告诉用户：

1. 发生了什么。
2. 库存发生了怎样的变化。
3. 是否可以撤销。

### 3.4 离线可靠

- 所有核心库存操作必须支持离线。
- 库存变化和操作记录必须在同一 IndexedDB 事务内完成。
- 不允许静默吞掉写入或导入错误。

## 四、信息架构

底部导航调整为：

1. 首页
2. 库存
3. 操作记录
4. 设置

### 4.1 首页

快速操作、核心库存数字、今日记录和库存脉搏。

### 4.2 库存

统一承载两种库存，通过顶部分段切换：

- 原盒养护
- 散养库存

### 4.3 操作记录

展示入库、出库、拆盒、价格调整、删除和恢复历史。

### 4.4 设置

承载 XLSX 导入导出、备份状态、密码、安全设置和未来柜号管理。

## 五、首页

### 5.1 顶部

- 当前日期。
- 简短问候语。
- 最近备份状态。

备份提醒规则：

- 7 天内：已备份。
- 8–14 天：建议备份。
- 超过 14 天：突出提醒。

### 5.2 库存总览

首页只展示：

- 原盒总盒数。
- 库存总支数。
- 库存自定义总价值。

### 5.3 快速操作

四个固定入口：

- 入原盒。
- 入散支。
- 出库。
- 拆盒。

点击后使用底部操作面板，完成后返回首页并展示结果与撤销入口。

### 5.4 今日操作记录

按时间倒序展示：

- 操作类型。
- 品牌与型号。
- 数量变化。
- 柜号。
- 时间。
- 相关价值。

### 5.5 库存脉搏

展示相较昨天或上次打开的变化：

- 原盒数量变化。
- 散支数量变化。
- 使用中柜号数。
- 库存价值变化。

## 六、库存模式

### 6.1 原盒养护

- 按盒管理，一盒一条记录。
- 同款多盒仍然分别管理。
- 没有编号时使用系统内部唯一 ID。

字段：

| 字段 | 要求 |
|---|---|
| 品牌 | 必填 |
| 型号 | 必填 |
| 年份 | 必填，单个具体年份 |
| 每盒支数 | 必填 |
| 柜号 | 必填 |
| 每盒自定义价格 | 必填，可修改 |
| 编号 | 可选 |
| 来源/渠道 | 可选 |
| 环径 | 可选 |
| 长度（mm） | 可选 |
| 入库时间 | 默认当前时间，可修改 |

主要操作：编辑、拆盒、出库、删除。

### 6.2 散养库存

- 按支管理。
- 品牌、型号、年份和柜号都相同时自动合并支数。
- 不同柜号必须作为独立库存。
- 合并时保留现有单支价格，不用新价格覆盖。

字段：

| 字段 | 要求 |
|---|---|
| 品牌 | 必填 |
| 型号 | 必填 |
| 年份 | 必填，单个具体年份 |
| 支数 | 必填 |
| 柜号 | 必填 |
| 每支自定义价格 | 必填，可修改 |
| 入库时间 | 默认当前时间，可修改 |

主要操作：增加入库、按支出库、盘点、修改价格、删除。

### 6.3 柜号

- 2.0 继续使用自由输入。
- 未来升级为可维护的柜号列表。
- Excel 导入时只保留第一个连字符前的主柜号，例如 `M-R1-A` 转换为 `M`。

## 七、库存查询

统一搜索范围：

- 品牌。
- 型号。
- 年份。
- 柜号。
- 编号。

筛选条件：

- 库存类型。
- 品牌。
- 年份。
- 柜号。

2.0 第一阶段不实现复杂排序。

## 八、核心操作流程

### 8.1 原盒入库

1. 选择或输入品牌与型号。
2. 填写年份、每盒支数、柜号和自定义价格。
3. 可选填写编号、来源、环径和长度。
4. 确认入库。

多盒入库时按盒生成独立记录。

### 8.2 散支入库

1. 选择品牌、型号和年份。
2. 填写支数、柜号和单支价格。
3. 系统检查同款同柜号散支。
4. 展示合并结果并确认。

合并时增加支数并保留现有价格。

### 8.3 出库

1. 选择原盒或散养。
2. 选择目标库存。
3. 填写数量、原因、时间和备注。
4. 确认出库。

出库原因固定为：自用、赠送、出售、其他。

### 8.4 拆盒

1. 选择原盒。
2. 展示盒内支数与盒价。
3. 自动计算散支单价。
4. 允许手动修改散支单价。
5. 选择目标柜号。
6. 整盒转入散养。

拆盒不支持只转移部分后继续按原盒管理。

## 九、价格与价值

价格是用户自定义库存计价，不代表买入成本。

- 原盒价值 = 每盒自定义价格。
- 散养价值 = 每支自定义价格 × 当前支数。
- 总价值 = 原盒价值 + 散养价值。

每次价格修改必须保存：

- 修改前价格。
- 修改后价格。
- 修改时间。
- 操作对象。

## 十、操作记录

记录类型：

- 原盒入库。
- 散支入库。
- 原盒出库。
- 散支出库。
- 拆盒。
- 价格修改。
- 删除。
- 恢复。

每条记录保存：

- 操作前快照。
- 操作后快照。
- 数量变化。
- 价格变化。
- 柜号。
- 操作时间。
- 备注。

记录页支持按日期、类型和关键词筛选。

## 十一、删除、恢复与撤销

- 库存允许直接删除。
- 删除前展示库存摘要和影响。
- 删除后生成完整删除记录。
- 操作完成后提供短时撤销。
- 记录页提供长期恢复。
- 恢复散养库存时重新检查是否需要合并。

## 十二、XLSX 导入与备份

### 12.1 导入

支持原有工作表：

- `原盒`
- `散支`
- `入库`
- `出库`

导入预览必须展示：

- 新增原盒数量。
- 新增散养数量。
- 历史记录数量。
- 重复数量。
- 异常数量与原因。

转换规则：

- `52|135` 拆分为环径 52、长度 135 mm。
- 年份范围取最早年份。
- 层级柜号取主柜号。
- `/` 转换为空值。
- 重复数据默认跳过。

失败时必须明确展示错误，并保证事务整体回滚。

### 12.2 备份

- 导出全部库存与操作记录为 XLSX。
- 支持从系统备份恢复。
- 记录最近一次成功导出时间。
- 清除本地数据前提醒先备份。

## 十三、密码与隐私

- 使用 6 位本地数字密码。
- 密码以 PBKDF2 加盐哈希保存。
- 离开应用超过 5 分钟自动锁定。
- 密码和库存不上传服务器。
- 忘记密码只能清除本地数据。

## 十四、视觉系统

### 14.1 色彩

- 主背景：温暖浅象牙白。
- 主色：森林绿。
- 辅助面：浅鼠尾草绿。
- 强调色：低饱和黄铜色。
- 危险色：克制暗红色。

黄铜色只用于价值、拆盒转移和备份提醒。

### 14.2 排版

- 中文系统字体优先。
- 正文 14–16px。
- 数字使用清晰的表格数字特性。
- 全局不超过两套字体。

### 14.3 组件

- 优先使用留白、分组和分隔线。
- 减少满屏卡片和卡片嵌套。
- 列表使用统一表面与行分隔。
- 阴影仅用于底部面板和临时浮层。
- 图标统一采用专业线性图标，不使用 Emoji。

## 十五、必须覆盖的状态

- 初次使用。
- 空库存。
- 正常数据。
- 搜索无结果。
- 导入处理中、成功、部分异常和失败。
- IndexedDB 写入失败。
- 离线。
- 自动锁定。
- 删除确认。
- 操作成功与可撤销。

所有错误提示必须说明：发生了什么、数据是否受影响、下一步怎么办。
