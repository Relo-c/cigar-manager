# Cigar Manager 项目计划

> 状态：第一版需求已确认，尚未开始开发  
> 产品阶段：个人测试版，为未来多人使用预留扩展能力

## 一、最高优先级协作准则

以下准则适用于本项目中的所有操作：

1. 在执行任何操作前，必须先通过提问确认需求；只有在对用户需求的理解达到至少 95% 把握后，才能执行相应操作。
2. 问题必须一个一个地问。每次只提出一个问题，收到用户回答后，才能提出下一个问题。
3. “任何操作”包括但不限于：读取文件、搜索内容、检查状态、修改或删除文件、运行命令或程序，以及执行会影响外部状态的操作。

## 二、项目定位

Cigar Manager 是一款面向 iPhone 的中文雪茄库存管理 H5/PWA。

第一版仅供单用户测试使用，数据保存在 iPhone 本地；产品和数据结构应为未来增加多用户、云端同步与照片功能预留扩展空间。

## 三、第一版使用方式

- 使用 iPhone Safari 打开 HTTPS 网站。
- 支持添加到 iPhone 主屏幕，以接近原生 App 的方式启动。
- 首次成功加载后，尽量支持完全离线使用。
- 使用 GitHub Pages 进行静态网站托管。
- 项目需要重新初始化 Git，并创建新的 GitHub 仓库。
- 第一版仅提供中文界面。
- 视觉风格采用简洁、明亮的现代管理工具风格。

## 四、库存模式

### 4.1 原盒养护

- 按盒管理。
- 严格执行“一盒一条库存记录”。
- 即使品牌、型号、年份和柜号相同，多盒也必须分别管理。
- 没有雪茄编号时，由系统生成内部唯一标识。

原盒字段：

| 字段 | 要求 |
|---|---|
| 品牌 | 必填 |
| 型号 | 必填 |
| 年份 | 必填，仅支持单个具体年份 |
| 每盒支数 | 必填 |
| 柜号 | 必填，第一版自由输入 |
| 每盒自定义价格 | 必填或由导入数据提供，可修改 |
| 编号 | 可选 |
| 来源/渠道 | 可选 |
| 环径 | 可选 |
| 长度（mm） | 可选 |
| 入库时间 | 默认当前时间，可手动修改或补录 |

### 4.2 散养

- 按支管理。
- 品牌、型号、年份和柜号均相同的散支多次入库时，自动合并支数。
- 同款散支位于不同柜号时，必须作为多条独立库存管理。
- 合并库存时保留原有单支自定义价格，不用新入库价格覆盖。
- 用户可随时手动修改已有散支库存的单支自定义价格。

散养字段：

| 字段 | 要求 |
|---|---|
| 品牌 | 必填 |
| 型号 | 必填 |
| 年份 | 必填，仅支持单个具体年份 |
| 支数 | 必填 |
| 柜号 | 必填，第一版自由输入 |
| 每支自定义价格 | 必填或由拆盒/导入数据生成，可修改 |
| 入库时间 | 默认当前时间，可手动修改或补录 |

### 4.3 柜号规则

- 第一版使用自由输入文本。
- 未来预留升级为“柜号列表管理与选择”的能力。
- 从现有 Excel 导入层级柜号时，仅保留第一个连字符前的主柜编号。
- 示例：`M-R1-A` 转换为 `M`，`A-R2-B` 转换为 `A`。

## 五、价格与价值规则

- 系统中的价格是用户自定义的库存计价，不代表实际买入成本。
- 原盒以“每盒自定义价格”计价。
- 散养以“每支自定义价格”计价。
- 自定义价格允许随时手动修改。
- 修改价格后，库存总价值及相关统计自动重新计算。
- 每次价格修改都要记录：修改前价格、修改后价格和修改时间。

计算规则：

- 原盒库存价值 = 每盒自定义价格。
- 散养库存价值 = 每支自定义价格 × 当前支数。
- 全部库存价值 = 所有原盒库存价值 + 所有散养库存价值。

## 六、拆盒规则

- 拆盒必须整盒操作，不支持只拆部分后继续按原盒管理。
- 拆盒后，原盒库存减少一盒，即移除被拆的原盒记录。
- 盒内全部支数转入散养库存。
- 默认散支单价 = 原盒每盒自定义价格 ÷ 每盒支数。
- 拆盒时允许手动修改自动计算出的散支单价。
- 如果转入的散支与现有散养库存的品牌、型号、年份和柜号相同，则自动合并支数，并保留现有散支单价。
- 拆盒操作必须写入操作记录。

## 七、入库与出库

### 7.1 入库

- 支持原盒入库和散支入库。
- 入库时间默认使用当前日期和时间。
- 允许手动修改时间，以便补录历史记录。
- 每次入库必须写入操作记录。

### 7.2 出库

- 原盒和散养库存都支持出库。
- 固定出库原因：自用、赠送、出售、其他。
- 出库时间默认使用当前日期和时间。
- 允许手动修改时间，以便补录历史记录。
- 每次出库必须写入操作记录。

## 八、删除与恢复

- 原盒和散养库存都允许直接删除，不强制通过出库减少库存。
- 删除前应提供明确确认。
- 删除后自动生成“删除记录”，保存被删除库存的完整信息和删除时间。
- 支持从删除记录恢复库存。
- 恢复库存的操作也应写入操作记录。

## 九、操作记录

提供独立的操作记录页面，集中查看：

- 入库记录
- 出库记录
- 拆盒记录
- 价格修改记录
- 删除记录
- 恢复记录

操作记录允许手动修改和删除。

## 十、首页与库存查询

### 10.1 首页概况

应用打开后首先进入统计概览，展示：

- 原盒总盒数
- 原盒总支数
- 散养总支数
- 全部库存自定义总价值
- 按品牌统计的库存数量分布图表
- 按品牌统计的库存自定义价值分布图表

### 10.2 库存列表

- 原盒与散养库存分开查看。
- 支持按品牌、型号、年份和柜号搜索或筛选。
- 第一版不提供排序功能。

## 十一、Excel 导入、备份与恢复

### 11.1 现有测试数据导入

第一版需要直接兼容项目中的 `testData.xlsx`，导入以下工作表：

- `原盒`
- `散支`
- `入库`
- `出库`

导入规则：

- 保留 Excel 中的自定义价格和价值信息，不将其视为买入成本。
- `52|135` 形式的数据拆分为环径 `52` 和长度 `135 mm`。
- 年份范围取最早年份，例如 `2021-2023` 转换为 `2021`。
- 多层级或组合柜号仅保留主柜编号，例如 `M-R1-A & M-R1-C` 转换为 `M`。
- Excel 中使用 `/` 表示缺失的可选字段时，导入为空值。
- 导入前提供数据预览和确认，避免重复导入或错误覆盖。

### 11.2 备份与恢复

- 支持将全部本地数据导出为 XLSX 文件。
- 支持从系统导出的 XLSX 文件恢复全部数据。
- 备份应覆盖库存、操作记录和必要的应用设置。

## 十二、本地数据与访问保护

- 使用 IndexedDB 保存库存、操作记录和应用设置。
- 不使用云端数据库，不在设备间同步。
- 提供本地数字密码解锁。
- 第一版不接入 Face ID。
- 忘记密码时，允许清除全部本地数据并重新初始化。
- 应清楚提示用户：删除 Safari 网站数据或卸载相关网站数据可能导致库存丢失，应定期导出 XLSX 备份。

## 十三、PWA 与技术方向

第一版采用纯前端静态架构，已确认技术栈：

- Vue 3 + TypeScript：应用框架与类型系统。
- Vite：开发和构建工具。
- Vant：移动端 UI 组件库。
- Pinia：应用状态管理。
- Dexie.js：IndexedDB 数据访问、事务与版本迁移。
- ECharts：库存与价值横向条形图。
- SheetJS：XLSX 导入、导出、备份与恢复。
- Vite PWA 插件：生成 Web App Manifest、Service Worker 和离线缓存。
- 响应式移动端 H5，优先适配 iPhone Safari。
- 静态资源部署至 GitHub Pages，并使用 HTTPS。

技术实现必须避免把未来多用户能力锁死。后续可增加后端 API、账户体系、云数据库和跨设备同步，但第一版不实现这些功能。

## 十四、页面导航与交互

### 14.1 底部导航

底部使用四个主要入口：

1. 概况
2. 原盒
3. 散养
4. 记录

导入、导出、密码和其他设置放在“更多”菜单中。

### 14.2 新增入口

- 原盒页面使用醒目的“＋”按钮新增原盒库存。
- 散养页面使用醒目的“＋”按钮新增散支库存。

### 14.3 首页图表

- 品牌库存数量分布采用横向条形图。
- 品牌库存自定义价值分布采用横向条形图。

## 十五、密码策略

- 使用 6 位数字密码。
- 应用从后台恢复时，若离开时间超过 5 分钟，则要求重新解锁。
- 忘记密码时，只能通过清除全部本地数据重新初始化。

## 十六、重复导入策略

- 导入 XLSX 时识别库存与操作记录中的重复数据。
- 默认跳过识别出的重复记录。
- 正式写入 IndexedDB 前展示导入预览，包括新增、跳过和异常数据数量。
- 只有用户确认预览结果后才执行导入。

## 十七、Git 与部署

- 本地重新初始化 Git 仓库。
- GitHub 仓库名称：`cigar-manager`。
- GitHub 仓库可见性：私有。
- 使用 GitHub Pages 部署 HTTPS 静态网站。
- 项目中不得存放密码、访问令牌或其他秘密信息。

## 十八、第一版暂不实现

- 多用户注册与登录
- 云端存储与跨设备同步
- 照片上传和雪茄图片库
- Face ID 解锁
- 详细柜内层位管理
- 预设柜号列表管理
- 库存列表排序
- 买入成本和利润统计

## 十九、建议实施阶段

1. 初始化前端项目、Git 仓库和基础 PWA。
2. 建立 IndexedDB 数据模型与本地密码机制。
3. 实现原盒和散养库存的增、改、删、查。
4. 实现拆盒、入库、出库、删除与恢复流程。
5. 实现操作记录和价格变更追踪。
6. 实现首页统计概览、品牌分布图表和库存筛选。
7. 实现 `testData.xlsx` 导入。
8. 实现 XLSX 备份与恢复。
9. 完成 iPhone 适配、离线测试和主屏幕安装测试。
10. 创建 GitHub 仓库并部署到 GitHub Pages。

第一版开发所需的产品、数据、技术、交互与部署方向均已确认。开始任何实现操作前，仍须遵守最高优先级协作准则，逐次确认具体执行范围。
