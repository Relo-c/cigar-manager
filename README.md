# Cigar Manager

面向个人收藏者的中文雪茄库存管理 PWA。当前正式版本为 **v1.0 H5 内部储存版**，优先适配 iPhone Safari，支持添加到主屏幕和离线访问。

- 在线地址：<https://relo-c.github.io/cigar-manager/>
- v1.0 Release：<https://github.com/Relo-c/cigar-manager/releases/tag/v1.0>
- 数据模式：单用户、本机 IndexedDB 存储，不上传库存、密码或备份文件

## 产品范围

应用包含四个主要页面：

1. **首页**：库存概览、快速操作、今日操作记录和库存脉搏。
2. **库存**：原盒与散养库存、搜索、品牌/柜号筛选及筛选后数量与价值汇总。
3. **操作记录**：入库、出库、拆盒、价格修改和删除的历史快照。
4. **设置**：XLSX 导入、模板下载、备份导出、iPhone 安装说明和本地密码信息。

## 已实现功能

### 库存管理

- 原盒按盒管理，一盒一条记录；同款多盒分别保存。
- 散养按支管理；品牌、型号、年份和柜号相同时自动合并。
- 原盒和散养均支持新增、编辑、出库和删除。
- 原盒支持整盒拆为散养，以及整盒或按支出库。
- 搜索覆盖品牌、型号、年份和柜号。
- 品牌与柜号支持组合单选筛选，数量、支数和总价值随结果实时变化。
- 金额使用完整人民币数值和千分位展示，不使用“万/亿”缩写；长金额会自适应字号。

### 品牌与录入联想

- 内置 27 个 Habanos 古巴雪茄品牌 Logo。
- 维护标准中文品牌名、英文名和常见中文别名。
- 品牌支持中文、英文和别名联想，选中后统一保存为标准中文名。
- 型号按所选品牌联想历史记录。
- 年份、柜号、环径、长度、支数和价格等字段可复用已有库存数据；新录入数据自动成为后续建议来源。

### 操作记录

- 记录入库、出库、拆盒、价格修改和删除。
- 每条记录保存操作时间、库存类型、品牌、型号、年份、柜号、数量、当时价格及可选备注。
- 出库原因包括自用、赠送、出售和其他。
- 删除需要二次确认，删除后不可恢复，但完整库存快照保留在操作记录中。

### XLSX 与备份

- 支持选择 XLSX、解析预览、重复检查、异常跳过和事务化导入。
- 导入失败时整批回滚，不产生部分库存。
- 数据管理窗口可直接下载 `雪茄库存导入模板.xlsx`。
- 可导出包含原盒、散支和操作记录的完整 XLSX 备份。
- `testData.xlsx` 不属于公开站点或构建产物。

### PWA、密码与隐私

- 支持 iPhone Safari“添加到主屏幕”。
- Service Worker 自动更新，核心静态资源支持离线访问。
- 使用 6 位本地数字密码；密码经 PBKDF2 加盐哈希保存。
- 离开应用超过 5 分钟后自动锁定。
- 忘记密码只能清除本机应用数据，因此应定期导出 XLSX 备份。
- 输入控件针对 iPhone 使用不小于 16px 的字号，避免聚焦输入框时页面自动放大。

## 数据规则

### 原盒库存

必填字段：

- 品牌
- 型号
- 年份（精确到年）
- 每盒支数
- 柜号
- 自定义盒价
- 入库时间

可选字段：编号、来源、环径、长度。

### 散养库存

必填字段：

- 品牌
- 型号
- 年份（精确到年）
- 支数
- 柜号
- 自定义单支价格
- 入库时间

合并已有散养记录时增加支数并保留原有单支价格；价格可在编辑库存时主动修改。

### 价值计算

- 原盒价值 = 自定义盒价
- 散养价值 = 自定义单支价格 × 当前支数
- 库存总价值 = 全部原盒价值 + 全部散养价值

价格仅用于用户自定义库存计价，不代表买入成本，也不计算利润。

## 出库、拆盒与删除规则

### 拆盒

1. 从原盒库存移除该盒。
2. 按盒内支数转入同品牌、型号、年份和柜号的散养库存。
3. 新散养记录的默认单支价格为“原盒价格 ÷ 每盒支数”。
4. 如果匹配到已有散养记录，则合并数量并保留已有散养价格。
5. 写入拆盒操作记录。

### 出库

- 原盒可选择整盒出库或按支出库。
- 整盒出库后直接移出当前原盒库存。
- 原盒按支出库时自动拆盒，出库指定支数，剩余支数转入散养。
- 散养支持部分出库；剩余数量继续保留。
- 全部出库后移出当前库存，但操作记录永久保留。

### 删除

- 原盒和散养均需二次确认。
- 删除不计为出库，不可恢复。
- 删除记录保存被删除库存的完整快照。

## XLSX 导入格式

### `原盒` 工作表

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 品牌 | 是 | 支持标准中文名、英文名或已维护别名 |
| 型号 | 是 | 文本 |
| 环径 | 否 | 使用 `52\|135` 格式，左侧为环径，右侧为长度 mm |
| 单盒价格 | 否 | 数字，缺省为 0 |
| 单盒支数 | 是 | 正整数 |
| 编号 | 否 | 有编号时优先用于重复判断 |
| 年份 | 是 | 四位年份；含范围时取最早年份 |
| 来源 | 否 | 文本 |
| 柜号 | 是 | 层级柜号只取开头主柜号，例如 `M-R1-A` 取 `M` |

### `散支` 工作表

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 品牌 | 是 | 品牌名称 |
| 型号 | 是 | 文本 |
| 单支价格 | 否 | 数字；也兼容“单盒价格 ÷ 单盒数量” |
| 支数 | 是 | 正整数 |
| 年份 | 是 | 四位年份 |
| 柜号 | 是 | 柜号文本 |

兼容旧备份中的 `入库`、`出库` 和 `操作记录` 工作表。单元格 `/` 视为空值；重复数据默认跳过。

## 本地开发

要求：Node.js 与 pnpm。

```bash
pnpm install
pnpm run dev
```

生产构建：

```bash
pnpm run build
```

Vite 的部署基路径为 `/cigar-manager/`。推送到 `main` 后，由 `.github/workflows/deploy-pages.yml` 构建并部署 GitHub Pages。

## iPhone 安装

1. 使用 iPhone Safari 打开 <https://relo-c.github.io/cigar-manager/>。
2. 点击 Safari 底部“分享”。
3. 选择“添加到主屏幕”。
4. 点击右上角“添加”。

PWA 更新后，如桌面版本仍显示旧界面，可完全关闭应用后重新打开；必要时先在 Safari 中刷新在线地址。

## 视觉与验收基线

- 冷调、简洁、明亮的雾紫蓝柔光渐变风格。
- 中文优先使用苹方/系统无衬线字体。
- 主操作使用紫蓝渐变与白色文字；禁用状态仍需保持文字清晰。
- 主要验收视口为 `390 × 844`，不得出现横向滚动或关键文字裁切。
- 库存数量、支数和完整金额在窄屏上必须可读。
- 搜索图标、占位文字与搜索框保持垂直居中。
- 核心库存变化与操作记录必须在同一 IndexedDB 事务中完成。
- 每次功能修改后至少执行 `pnpm run build`，涉及界面时再进行真实移动视口检查。

## 当前不包含

v1.0 暂不包含多用户、云同步、多设备同步、Face ID、照片管理、湿度监测、买入成本与利润、详细柜内层位、通知推送和复杂权限系统。

未来如扩展多人模式，应在数据模型中引入用户、工作区、操作人、同步状态和服务端版本等字段，并设计权限与冲突解决机制。
